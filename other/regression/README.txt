Deterministic regression assets for Noxim.

Files:
- configs/: pinned base YAML configurations used by the regression runner
- cases.txt: curated regression matrix
- expected/: normalized golden outputs committed for each case

Usage:
1. From the repository root, run ./regression.sh
2. The script builds bin/noxim, runs the configured cases, normalizes the
   stable summary section of the simulator output, and compares it with
   other/regression/expected/

Maintaining baselines:
1. Review and update the regression matrix in other/regression/cases.txt
2. Regenerate golden outputs with ./regression.sh --update
3. Rerun ./regression.sh to verify the refreshed baselines

Notes:
- The regression YAML files pin rnd_generator_seed so each case is
  deterministic without requiring a -seed command-line override.
- TABLE_BASED cases use temporary XY-equivalent routing tables generated by
  regression.sh. Those scratch files are written under other/regression/generated/.
